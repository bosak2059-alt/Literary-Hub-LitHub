# Шаг 1. Определение темы проекта и концепции

## Тема проекта
1) WEB-приложение
2) Платформа для книжных клубов
3) Literary Hub | LitHub
Сайт, где пользователи создают или присоединяются к книжным клубам, обсуждают прочитанное, ставят оценки, делятся мнениями.

## Основные цели проекта
1) Основные цели приложения:
    1.1 Объединить читателей с общими интересами.
    1.2 Упростить создание и ведение книжных клубов.
    1.3 Обеспечить удобное взаимодействие между участниками.
    1.4 Стимулировать регулярное чтение через сообщество и поддержку.

2) 
Целевая аудитория:
    1.1 Любители чтения от 16 до 45 лет.
    1.2 Студенты, молодые специалисты, педагоги, библиотекари.
    1.3 Люди, ищущие интеллектуальное общение и новые книги.

Решаемая проблема:
    1.1 Сложность найти подходящий книжный клуб.
    1.2 Неудобство организации встреч и выбора книг.
    1.3 Отсутствие постоянного общения между сессиями.
    1.4 Нехватка мотивации читать регулярно.

3) Основные функции: 
    1.1 Управление клубами
        (a)__Создание клуба__: форма с указанием названия, жанра, формата встреч (онлайн/офлайн/асинхронно), частоты обсуждений, правил и настроек приватности.
        (b)__Вступление/Выход__: однокликовое присоединение с автоматическим обновлением счётчика участников; возможность покинуть клуб с сохранением истории в профиле.
        (c)__Каталог клубов__: поиск по названию/жанру, фильтрация по формату и активности, карточки с кратким описанием и рейтингом.
    
    1.2 Трекинг чтения и работа с книгами
        (a)__Текущая книга клуба__: отображение обложки, автора, объёма и даты следующей встречи.
        (b)__Прогресс-бар__: индивидуальное отслеживание прочитанного, визуальная индикация % выполнения.
        (c)__Подготовка к встрече__: возможность скачивать вопросы для дискуссии.
    
    1.3 Асинхронное общение
        (a)__Встроенный чат__: сохранение истории сообщений, поддержка цитирования, автоскролл, отправка по Enter.
        (b)__Режим «тихого участника»__: возможность читать обсуждения без обязательств публиковать сообщения (учитывает персону Дмитрия).
        (c)__Модерация контента__: базовые правила отображения, пометка удалённых сообщений, ограничение доступа для не-участников.

    1.4 Организация и планирование
        (a)__Система голосований__: выбор следующей книги, динамический пересчёт процентов, блокировка повторного голоса, прозрачная аналитика.
        (b)__Календарь встреч__: сетка событий с привязкой к клубам, подтверждение участия, экспорт в Google/Apple Calendar.
        (c)__Напоминания__: визуальные метки о приближающихся встречах и низком прогрессе чтения (готовится к интеграции с уведомлениями).

    1.5 Личный кабинет и персонализация
        (a)__Профиль пользователя__: аватар с инициалами, статистика (книги, клубы, рецензии, серия чтения), список активных клубов.
        (b)__История активности__: отображение последних рецензий, прочитанных книг и участия в голосованиях.

    1.4 **Что важно**: голосования, простой интерфейс.

__На основе анализа конкурентов можно сделать следующие выводы:__

1) Что можно позаимствовать:
    1.1 У __Goodreads__ — систему рейтингов и рецензий, а также идею персональных списков книг.
    
    1.2 У __Bookmate__ — подход к персонализированным подборкам и аккуратному визуальному стилю.
    
    1.3 У __Meetup__ — удобный календарь событий с напоминаниями и функцией подтверждения участия.
    
    1.4 У __Telegram__ — гибкость общения и поддержку цитирования сообщений.

    1.1 Устранить __фрагментацию общения__, объединив чат, календарь и голосования в одном месте.

    1.3 Добавить __режим “тихого участника”__, чтобы вовлечь более стеснительных читателей.
    
    1.4 Внедрить __аналитику голосований__ — диаграммы, кто за что голосовал, чтобы повысить прозрачность.
